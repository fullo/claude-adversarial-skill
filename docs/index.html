How It Works

A rigorous 6-step pipeline inspired by Chain-of-Verification research

0

Identify & Gather

Detect the verification domain, collect artifacts, and establish ground truth from tests, specs, and production state.

→
1

Decompose

Break every artifact into individual, verifiable claims. Each claim becomes a target for independent verification.

→
2

Adversarial Questions

Generate counter-questions for each claim and abstract findings into recurring failure categories.

→
3

Independent Verify

Trace actual code paths. Apply hidden behavior probing and the modular adversarial scaffold.

→
4

Structured findings with file:line references, quoted evidence, reproduction scenarios, and confidence scores.

→
5

Project Discovery

Propose updates to TODO.md, SPEC.md, PLAN.md, and CHANGELOG based on findings. Waits for user confirmation.

Core Techniques

Four research-backed methodologies working together

Chain-of-Verification

Decompose artifacts into verifiable claims, generate adversarial counter-questions, then verify each claim independently against ground truth. Prevents confirmation bias by separating generation from verification.

Dhuliawala et al., 2023

Abstractive Red-Teaming

Don't just find individual bugs — identify recurring failure categories across the codebase. Six patterns tracked: frequency assumptions, implicit ordering, stale state, missing completeness, silent fallthrough, assumed environment.

Anthropic, 2026

Hidden Behavior Probing

Four probing strategies that go beyond surface-level review: indirect probing (trace execution), scaffolded probing (chain findings), cross-reference probing (claims vs reality), and absence probing (what's NOT there).

AuditBench — Anthropic, 2026

Modular Adversarial Scaffold

Five-module framework: suspicion modeling (what would a normal reviewer miss?), attack selection (highest-risk claims first), plan synthesis, execution (actually read the code), and subtlety detection.

Anthropic, 2025
